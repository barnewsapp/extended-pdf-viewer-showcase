You can override the default function Window.ngxConsoleFilter(logLevel, message) to filter the log messages.

This live demo re-defines ngxConsolefilter() to grab the version number from the console messages and to store it in a variable so it can be shown here:

{{ version }}

The original use-case of ngxConsolefilter() is suppressing messages you don't want to log. For example, many PDF documents cause the viewer to emit messages like Warning: TT: undefined function: 32. That's one of the messages you can almost always safely ignore, so it doesn't make sense to pump it into, say, an ELK stack. If your cust om implementation of ngxConsolefilter() returns false depending on the message text, the message is never logged.

The live demo shows that, too: when you open the console window, you'll see every message with the exception of the version info.
