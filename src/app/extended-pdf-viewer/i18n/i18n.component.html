ngx-extended-pdf-viewer ships with translation for more than 100 languages and dialects. When you copy the node_modules/ngx-extended-pdf-viewer/assets/locale folder to the assets/locale folder of your application, you can select your language by setting the [language] attribute and by setting [useBrowserLocale]="true".

language default (usually English) es-ES (Spanish) de-DE (German) fr-FR (French) pt-PT (Portuguese) en-GB (English) el (Greek) nl-BE (Belgian) ar (arab)

Caveat: The attribute "language" is only read when the component is drawn initially. So this demo hides the PDF viewer for a second when you're switching from one language to another.

Do you need merely one or two languages? Set [useBrowserLocale]="false" and add the translation file to the index.html (or any other HTML file that's present in the DOM when ngx-extended-pdf-viewer initializes).

I've prepared HTML snippets for each language in the folder node_modules/ngx-extended-pdf-viewer/assets/inline-locale-files.

The locales are Json files. If you need a single language, you can simply copy the content of one of these files and put them into the index.html.

Caveat:The property default_locale of the JSON snippet is currently ignored. As of version 10.x, you have to set [language]="'<your-language-code>'"
