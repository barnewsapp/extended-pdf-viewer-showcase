This demo shows which events are emitted when a PDF file is rendered.

Note that after-load-complete is thrown before any page has been rendered. If you want to set the page number, you'll need to wait a bit longer.

There's also the event `on-error`. If you're using the minified version of the pdf.js worker thread (which is the default), the parameter of the event is missing. The slow non-minified version reports the precise cause of the error in the parameter.
